1. Introduction

One very common Java interview question is:

What is the difference between Fail-Fast and Fail-Safe iterators?

This question is usually asked when discussing:

  • ArrayList
  • HashMap
  • ConcurrentHashMap
  • CopyOnWriteArrayList
  • Iterators
  • ConcurrentModificationException

Understanding this properly requires understanding how iterators work internally.

2. The Core Problem — Modifying Collection While Iterating

Consider this example:

List<Integer> list = new ArrayList<>();
list.add(1);
list.add(2);
list.add(3);

for (Integer i : list) {
    if (i == 2) {
        list.remove(i);
    }
}

This will throw:

ConcurrentModificationException

But why?

Because the collection was modified while iteration was in progress.

This is where Fail-Fast behavior comes in.

3. Fail-Fast Iterator

Definition

A Fail-Fast iterator immediately throws a
ConcurrentModificationException
if the collection is structurally modified after the iterator is created.

Structural Modification Means

  • Adding element
  • Removing element
  • Clearing collection
  • Resizing internal structure

Not structural:

  • Updating value in existing entry (e.g., map.put(key, newValue))

4. How Fail-Fast Works Internally

Most Java collections maintain a variable called:

modCount

Every time the collection structure changes:

modCount++

When iterator is created:

expectedModCount = modCount

During iteration:

if (modCount != expectedModCount)
    throw ConcurrentModificationException

Pseudo Code

if (modCount != expectedModCount) {
    throw new ConcurrentModificationException();
}

This check happens during:

  • next()
  • remove()
  • hasNext()

So the iterator fails fast as soon as modification is detected.

5. Collections With Fail-Fast Iterators

These collections use Fail-Fast iterators:

  • ArrayList
  • LinkedList
  • HashMap
  • HashSet
  • TreeMap
  • TreeSet

Basically:

Most collections from java.util are Fail-Fast.

6. Fail-Safe Iterator

Definition

A Fail-Safe iterator does not throw ConcurrentModificationException
because it iterates over a copy of the collection instead of the original collection.

So modifications are allowed while iterating.

This is also called:

Weakly consistent iterator

7. Collections With Fail-Safe Iterators

These collections use Fail-Safe iterators:

  • ConcurrentHashMap
  • CopyOnWriteArrayList
  • CopyOnWriteArraySet

These are from:

java.util.concurrent

8. Example — Fail-Safe

List<Integer> list = new CopyOnWriteArrayList<>();
list.add(1);
list.add(2);
list.add(3);

for (Integer i : list) {
    if (i == 2) {
        list.remove(i);
    }
}

This will NOT throw exception.

Because iterator is working on a copy of the list.

9. Fail-Fast vs Fail-Safe — Summary Table

Feature Fail-Fast Fail-Safe
Throws ConcurrentModificationException Yes No
Iterates over Original collection Copy of collection
Allows modification during iteration No Yes
Performance Faster Slower
Memory Low Higher
Package java.util java.util.concurrent
Examples ArrayList, HashMap ConcurrentHashMap, CopyOnWriteArrayList

10. Important Interview Questions

Interview Question 1

Why is it called Fail-Fast?

Because iterator fails immediately when modification is detected.

Interview Question 2

Is Fail-Fast guaranteed?

No.
Fail-Fast behavior is best effort, not guaranteed, because it is not synchronized.

Interview Question 3

How to safely remove elements while iterating?

Use iterator remove:

Iterator<Integer> it = list.iterator();

while (it.hasNext()) {
    Integer val = it.next();
    if (val == 2) {
        it.remove();
    }
}

This will NOT throw exception.

11. Quick Interview Summary (Very Important)

If interviewer asks:

Explain Fail-Fast vs Fail-Safe

You should answer like this:

Fail-Fast iterators throw ConcurrentModificationException if the collection is structurally modified during iteration.
They work by maintaining a modCount and expectedModCount and comparing them during iteration.
Examples include ArrayList, HashMap, HashSet.

Fail-Safe iterators do not throw exception because they iterate over a copy of the collection.
Examples include ConcurrentHashMap and CopyOnWriteArrayList.

Fail-Fast is faster and memory efficient, while Fail-Safe is thread-safe but uses more memory.

This is a perfect interview answer.

12. One Line Difference (Remember This)

Fail-Fast → Iterator works on original collection and throws exception on modification.
Fail-Safe → Iterator works on copy of collection and does not throw exception.
