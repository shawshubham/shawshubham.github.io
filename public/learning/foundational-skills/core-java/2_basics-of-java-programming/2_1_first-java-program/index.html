2. Your First Program: HelloWorld

We start with the classic Hello, World! program ‚Äî the simplest way to confirm your Java setup works.

Step-by-Step Guide

Step 1 ‚Äî Open your IDE or text editor

You may use:

  • IntelliJ IDEA (recommended)
  • Eclipse
  • VS Code
  • Notepad / Sublime (for simple editing)

Step 2 ‚Äî Create a new file

Name the file:

HelloWorld.java

Important: In Java, the public class name must match the filename exactly.

Step 3 ‚Äî Write the code

See Code in Git Repo Practice Java Code Here 
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Code Breakdown

public class HelloWorld

Defines a class named HelloWorld. Every Java program must be inside a class.

public static void main(String[] args)

This is the entry point of every Java application.

  • public ‚Üí accessible to the JVM
  • static ‚Üí runs without creating an object
  • void ‚Üí no return value
  • main ‚Üí method name
  • String[] args ‚Üí command-line arguments

System.out.println(“Hello, World!”);

Prints a message to the console.

  • System ‚Üí built-in Java class
  • out ‚Üí output stream
  • println() ‚Üí prints with a newline

3. Compiling the Java Program

Open a terminal or Command Prompt, then navigate to the folder containing your file:

cd path/to/your/file

Compile using:

javac HelloWorld.java

This will generate:

HelloWorld.class

This .class file contains Java bytecode, which the JVM executes.

4. Running the Program

Run the compiled bytecode:

java HelloWorld

You should see:

Hello, World!

üéâ Congratulations! You’ve successfully run your first Java program.

5. Troubleshooting Common Issues

‚ùå Error: class HelloWorld is public, should be declared in a file named HelloWorld.java

Your filename must match the public class name exactly.

‚ùå ‘javac’ is not recognized

JAVA_HOME or PATH is not set correctly.

‚ùå NoClassDefFoundError

You may be running:

Instead of:

java HelloWorld

‚ùå Nothing prints?

Check:

System.out.printIn          // incorrect (capital I)
