LeetCode Problem: Median of Two Sorted Arrays

1. Problem Statement

You are given two sorted arrays nums1 and nums2 of sizes m and n.

Return the median of the two sorted arrays.

The overall runtime complexity must be O(log (m + n)).

2. Clarifying Questions (Interview Thinking)

Before attempting a solution, confirm:

  • Are both arrays sorted? ‚Üí Yes
  • Can arrays be empty? ‚Üí Yes
  • Are values duplicated? ‚Üí Yes
  • Is merging allowed? ‚Üí No (time constraint forbids it)
  • What defines the median?
    • odd total length ‚Üí middle element
    • even total length ‚Üí average of two middle elements

3. Why This Problem Is Hard

At first glance, the problem looks simple:

  • merge the arrays
  • find the median

However:

  • merging takes O(m + n) time
  • the problem explicitly demands O(log (m + n))

This forces a non-obvious approach.

4. Pattern Identification

This problem does not belong to:

  • Two Pointers
  • Sliding Window
  • Stack
  • BFS
  • Greedy

It belongs to:

Binary Search on Partition

You are not searching for a value ‚Äî
you are searching for a valid split across two sorted arrays.

5. High-Level Idea (No Code Yet)

The core idea is to:

  • split both arrays into left and right parts
  • ensure:
    • the left side contains half of the elements
    • all elements on the left ‚â§ all elements on the right

Once such a partition is found, the median is determined immediately.

The challenge is finding that partition efficiently.

6. What This Question Tests

This problem evaluates:

  • ability to reason about invariants
  • comfort with logarithmic search
  • precision under constraints
  • understanding of advanced problem categories

It is not about speed or memorization.

7. How You Should Practice This Problem

Recommended approach:

  1. First, read and understand the Binary Search on Partition pattern
  2. Try to reason about:
    • what makes a partition valid
    • how moving the cut affects validity
  3. Only then attempt implementation
  4. Revisit the problem later if needed

Struggling here does not indicate weak fundamentals.

Key Takeaway

This problem is about finding the correct partition, not merging arrays.

Recognizing that early is the real victory.
