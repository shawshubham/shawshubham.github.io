1. Git Commands with Description

Below are a few git commands kept here as notes, not website ready but to treat it more of notes:

Commands Descrition
git –version to check the version of the git bash window
git init to initialize empty git repository in the directory it create .git file which can be visualized using ls -a command
git status to check the status of the repository ( if executed before creating repository throws error message “not a git repository” )
git config –global user.name “Your Name” this command is to configure git with user name
git config –global user.email “your@email.com” this command is to configure git with user email
git commit creating a commit, suppose at the start if we try running the command it will show the branch name and a message saying “nothing to commit”
git add This will add the specified fields in the staging area from working directory
git add –all or git add. This will add all the changed files in the staging area from working directory
git commit -m “commit message” this command commits the code with the commit message
git reset HEAD or git reset HEAD . or git rm –cache to ustage the specific files use the first command and to unstage all the files at once we can use the below command
git reset HEAD~1 –soft It reverts the previous commit but not the changes, changes will be present in the file
git log It shows all the detailed logs of the previous commit. Recent being at the top, Although most of the time we will be using IDE, so that can show in a more graphical way.
git log –patch It shows in more details, like what were the changes done etc.

2. Pulling changes from master to branch

Step 1: To keep the changes in the branch git stash

step 2: To fetch the changes from master git fetch

step 3: Rebase master on branch git rebase origin/master

step 4: Putting changes back to our branch which we stashed earlier gti stash pop

3. Removing uncommitted local changes

Step 1: See your changes in local using command. git status

Step 2: Execute the below command to point to the head, all local changes will be deleted post this command. git reset HEAD –hard

Step 3: Confirm your changes are deleted git status
